Technetium

JavaScript • Python • Django • D3 • Bitbucket API

A Bitbucket data aggregation tool built using Django and Bitbucket's RESTful API.Features a centralized issue tracker, repository subscription manager, and reports/graphs which enumerate the commit/issue activity of each user in a subscribed repository.

Technetium's frontend is built using Twitter Bootstrap. The centralized issue tracker aggregates issue data from subscribed repositories, and has been sped up through the use of Python grequests . Showing more issues and filtering is done using AJAX for responsive design.

Senior Thesis: Smarthouse For Everyone

JavaScript • NodeJS • Arduino • iOS • C • Django • Python

A plug-an-play device for remote control of any device that is plugged into it. The motiviation behind this project is to allow home automation for the visually impaired and physically challenged.

Hardware is built using Arduino along with a Arduino Wi-fi Shield for receiving web requests. Backend server is built using Node.js and Express.js. Rooms can be configured using the iOS mobile application, and saved rooms are handled on the server using Mongoose. Requests from the iOS client to the server is handled using the node.js requests library. Lastly, a web application alternative for interacting with the smarthouse stack was built using Django .

Fire Emblem - Chronicles of the Abyss

JavaScript • ImpactJS • HTML5

A new game development project as part of the CCNY ACM's Game Development Club. The motivation of this project is to create a JavaScript remake of Fire Emblem - Blazing Sword, which is playable using only the browser and the mouse, as well as to push the boundaries and limitations of the ImpactJS Game Engine.

  • Custom Assets

El-Gamal Attack

Cryptography • Python

A Python project with explores the semantic secureness of pycrypto's implementation of El Gamal. Using the Chinese Remainder Theorem, we can determine that some of the information of a transmitted message M is leaked. This information is efficiently computable by an adversary, showing that pycrypo's El Gamal implementation is not semantically secure.
